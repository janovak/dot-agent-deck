- **Suppressed `Pending` Card Flicker During Long LLM Gaps**
  Cards no longer briefly flash to `Pending` and back to `Working` during normal mid-conversation LLM-thinking gaps. Two tightenings to the stall heuristic in `AppState::apply_pending_timeout`:
  1. **Default `pending.timeout_seconds` raised from 10 → 30.** The lower default produced false positives on reasoning-model gaps and slow Copilot CLI responses; 30 s comfortably exceeds typical silence between tool calls while still catching true stalls in under a minute. Existing user configs are unchanged; only the `Default::default()` value moves.
  2. **Two-strike confirmation gate.** A session must be observed stale on two consecutive `apply_pending_timeout` calls (≈one extra second of grace at the default 1 s tick) before the Working → Pending flip fires. Any new event resets the strike counter via `apply_event`, so a single late-arriving event during an LLM gap can no longer produce a one-render Pending flash. A genuinely stuck agent still flips within `timeout + ~1 s`.
