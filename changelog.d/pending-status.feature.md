- **New `Pending` Card Status for Stalled Agents**
  Cards now flip from `Working` to `Pending` (magenta, bold) when an agent has been working for at least `pending.timeout_seconds` (default 10s) without an active tool and without any new hook events arriving. This catches the common Copilot CLI pattern of "agent printed a multiple-choice prompt and is waiting on stdin" — there's no hook event for that, but the silent-with-no-active-tool combination is a reliable signal. The bell rings on the transition (disable with `dot-agent-deck config set bell.on_pending false`). `Pending` is distinct from `WaitingForInput` (red), which is reserved for the explicit permission-prompt flow. Set `pending.timeout_seconds = 0` to disable the heuristic entirely; tune the threshold via `dot-agent-deck config set pending.timeout_seconds <N>`.
