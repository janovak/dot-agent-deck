- **Card Stays "Working" While Background Subagents Are Running**
  When the parent agent kicks off subagents (e.g., you ask Copilot CLI to "go review this" and it dispatches a background reviewer) the card no longer slides to **Idle** the moment the parent's turn ends. The session tracks a per-session count of in-flight subagents based on `SubagentStart` / `SubagentStop` hook events; an `Idle` event (Copilot's end-of-turn `sessionEnd`) only flips the card to Idle when that count is zero. The card returns to Idle as soon as the last subagent reports `SubagentStop`. The `Pending` heuristic also respects the count — a session that's "Working" only because of background subagents won't be false-flagged as stalled.
