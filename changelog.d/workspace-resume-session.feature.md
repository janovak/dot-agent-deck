- **Workspaces now resume the actual agent conversation**
  Saved workspaces capture each pane's live agent `session_id` and `agent_type` so a `--workspace <name>` restart spawns `copilot --resume <id>` (or `claude --resume <id>`) instead of starting a fresh conversation. `--continue` inherits the same behavior since both flows share the same restore path. Only the bare-agent invocation pattern is rewritten — if you've customized a pane's launch command (extra flags, `npx`, wrapper scripts, etc.) the original command is preserved unchanged. OpenCode panes are unaffected. If multiple agents have run in the same pane since the last save, the most-recently-active conversation is resumed; on a `/clear` or in-pane agent restart, the workspace tracks the new session id automatically. Recorded session ids are preserved across saves even if no live agent is bound at save time, so a resumeable id you just opened isn't wiped when you exit before the agent finishes booting — same "id may go stale" tradeoff that session bookmarks already accept.
