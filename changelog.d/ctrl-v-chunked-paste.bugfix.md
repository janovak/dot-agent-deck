- **Fixed Ctrl+V Splitting Rich Pastes Like Teams Tables**
  Pasting content with embedded tables or other formatting (the classic case: a Teams chat message containing a table, copied into the agent) via **Ctrl+V** previously broke the paste into multiple pieces — the agent would receive the first part, auto-submit on an internal carriage return, then receive the rest into a fresh prompt. Plain text via Ctrl+V was unaffected, and right-click paste already worked. Both paths now behave the same: the Ctrl+V handler reads the full clipboard contents directly (via `arboard`) rather than relying on the per-`Event::Paste` payload that Windows Terminal can split into multiple events for rich content. A 500 ms same-content dedup window then collapses any follow-up paste events that are actually chunks of the same logical Ctrl+V, even if the renderer hangs briefly between them.
