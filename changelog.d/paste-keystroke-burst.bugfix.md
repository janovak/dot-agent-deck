- **Multi-line Paste Detection Without `Event::Paste`**
  Multi-line pastes into agent panes are now batched and treated correctly even on terminals that don't generate `Event::Paste` for clipboard-pastes (notably Windows Terminal in some configurations, which feeds the clipboard content into the input queue as individual key presses). The event drain loop now accumulates printable Char and Enter keystrokes into a per-frame burst buffer. When the buffer holds 2+ keys at flush time, dot-agent-deck infers a paste, strips trailing CR/LF so it doesn't auto-submit, and wraps the payload in bracketed-paste markers when the focused pane runs a known agent (`copilot`, `claude`, `opencode`). Single-keystroke bursts (normal typing) pass through unchanged. Non-burstable keys (Ctrl/Alt combos, arrows, function keys, etc.) flush any pending burst before being written, so paste content and editing keys always arrive in the right order.
