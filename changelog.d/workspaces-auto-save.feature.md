- **Workspaces Auto-Save on Every Pane Change**
  The default session file (`session.toml`) and any named workspace are now persisted after every pane open, close, or rename — not just on clean exit. The save runs once per outer event-loop iteration and only writes when the current snapshot differs from what's already on disk, so steady-state runs do zero disk I/O. Practical effect: if you close the terminal window without going through the `Ctrl+C` quit dialog, the workspace file still contains a recent snapshot (at most one frame behind, typically <16 ms). Previously, closing the terminal abruptly discarded all changes since the last clean exit.
