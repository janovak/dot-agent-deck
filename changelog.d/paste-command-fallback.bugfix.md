- **Paste Mode Diagnostic in Status Bar**
  Every clipboard paste into a focused pane now flashes a brief `Pasted N chars (bracketed)` or `Pasted N chars (raw)` status message at the bottom of the dashboard. The mode reflects whether dot-agent-deck wrapped the paste in `\x1b[200~ … \x1b[201~` markers (which keep embedded newlines as content) or sent the bytes raw (which makes the child app interpret each newline as Enter). This makes it possible to diagnose multi-line paste issues at a glance: if the status says "bracketed" but the paste still gets split into separate submissions, the agent CLI is the culprit — it is not honoring bracketed-paste markers.
- **Force Bracketed Paste from Launch Command When Agent Hooks Are Absent**
  When a pane is launched with a known agent CLI command (`copilot`, `claude`, `opencode`, including `.exe`/`.cmd`/`.bat`/`.ps1` variants and quoted Windows paths), dot-agent-deck now wraps pastes in bracketed-paste markers immediately — without waiting for the agent to fire its first hook event. Previously, the override depended on `SessionState::agent_type` being set, which only happens after the first hook arrives. Pastes that happened during the early window (right after spawn, or when hooks were never installed) fell through to the raw branch and got submitted line-by-line.
